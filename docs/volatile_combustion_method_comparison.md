## 1. 当前实现状态

### 1.1 Python 实际逻辑（cell.py L264-289）

```python
# 燃烧区判据：pO2 > 0.05 atm（与 Fortran Line 345 一致）
is_combustion_zone = pO2_Pa > threshold_Pa

if is_combustion_zone:
    # 燃烧区：挥发分瞬时燃烧（无动力学）
    r_CH4, r_CO, r_H2 = self._instant_volatile_combustion(avail)
    r_homo['CH4_Ox'] = r_CH4
    r_homo['CO_Ox'] = r_CO
    r_homo['H2_Ox'] = r_H2
else:
    # 气化区：挥发分不氧化
    r_homo['CH4_Ox'] = r_homo['CO_Ox'] = r_homo['H2_Ox'] = 0.0
```

**结论**：Python 在燃烧区**已采用 Fortran 式瞬时完全燃烧**，`KineticsService.calc_homogeneous_rates` 计算的 CO_Ox/H2_Ox/CH4_Ox 在燃烧区内被**完全覆盖**，动力学参数（A、E）不参与放热计算。

## 3. 放热影响评估

### 3.1 当前实现（瞬时燃烧）

- **燃烧区**：与 Fortran 一致，放热 = 消耗量 × LHV
- **气化区**：挥发分不氧化，放热 = 0
- **净效应**：与 Fortran 在燃烧区的放热逻辑一致

### 3.2 若改用纯动力学方法

动力学速率：`r = A·exp(-E/RT)·C_A·C_B·V`（mol/s）

| 因素 | 对放热的影响 |
|------|--------------|
| **τ 短** | 单格停留时间短 → 动力学可能无法完全燃尽 → 放热减少 |
| **T 低** | k 小 → 燃烧慢 → 放热减少 |
| **T 高** | k 大 → 燃烧快 → 接近瞬时，放热接近 |
| **C 低** | 稀释 → r 小 → 放热减少 |

**典型 Cell 0 量级估算**（Paper_Case_6）：
- T ≈ 2000 K，P ≈ 4 MPa，V ≈ 0.5 m³
- C_CH4 ~ 0.01 kmol/m³，C_O2 ~ 0.01 kmol/m³
- k_CH4 = 1.6e10 × exp(-125600/(8.314×2000)) ≈ 8.5e6 m³/(kmol·s)
- r_CH4 ≈ 8.5e6 × 0.01 × 0.01 × 0.5 × 1000 ≈ 425 mol/s

挥发分 CH4 约 70 mol/s，动力学速率远大于可用量 → 在高温、高浓度下动力学也会接近“瞬时燃尽”，与瞬时燃烧差异不大。

### 3.3 差异主要出现在以下情况

| 工况 | 瞬时燃烧 | 动力学 | 放热差异 |
|------|----------|--------|----------|
| 燃烧区、高温、O2 充足 | 完全燃尽 | 接近完全燃尽 | 小 |
| 燃烧区、低温起燃 | 完全燃尽 | 部分燃尽 | 动力学放热更少 |
| 燃烧区、τ 极短 | 完全燃尽 | 可能未燃尽 | 动力学放热更少 |
| 气化区 | 0 | 0（当前也设为 0） | 无 |

---

## 4. 结论

1. **当前 Python 与 Fortran 在燃烧区一致**：均采用瞬时完全燃烧 + 燃烧焓，放热计算等价。
2. **动力学参数未参与燃烧区放热**：`CO_Ox`、`H2_Ox`、`CH4_Ox` 的 A、E 在燃烧区内被覆盖，仅影响诊断输出。
3. **若改为纯动力学**：在低温起燃或 τ 极短时，放热可能减少；在典型高温燃烧区，差异较小。
4. **建议**：维持当前瞬时燃烧实现，与 Fortran 一致；若需动力学控制，可增加开关（如 `UseKineticVolatileCombustion`）供对比研究。
